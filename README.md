# 💗 Study Circle

Study Circle is a peer-to-peer study aid designed for software engineering students.  
The platform allows students to connect with others who can offer help in specific subjects, creating a supportive academic community.

Many students cannot afford private tutoring, so this system provides a free and collaborative way to exchange knowledge and support.

---

## 🚀 Sprint 1 – Planning & Setup

In Sprint 1, we focused on preparing the project and setting up the team workflow.

This included:
- Refining the project idea
- Creating a Code of Conduct
- Developing personas
- Identifying ethical considerations
- Setting up our GitHub repository
- Creating a Kanban board
- Configuring the Docker development environment

---

## 📋 Sprint 2 – Requirements & System Design

In Sprint 2, we defined how the system should work before development began.

This included:
- Writing user stories
- Creating a Use Case Diagram
- Designing wireframes
- Producing activity diagrams

These helped guide the development in later sprints.

---

## 💻 Sprint 3 – Development

In Sprint 3, we built the core functionality of the application.

This included:
- Setting up Docker containers
- Creating and connecting the MySQL database
- Building pages using Express and PUG
- Users list page
- User profile page
- Study session listings and detail pages
- Tags/categories functionality
- Displaying real data from the database

---

## 🔥 Sprint 4 – Final Features

In Sprint 4, we improved the application and added more interactive features.

This included:
- User login system
- In-app messaging system
- Improved UI design and layout
- Better navigation between pages
- Overall refinement of the user experience

---

## ⚙️ CI/CD Pipeline (GitHub Actions)

This project uses GitHub Actions to implement a simple Continuous Integration (CI) pipeline.

When code is pushed to the main branch:
- The workflow runs automatically
- Dependencies are installed using npm install
- The application is checked to ensure it builds successfully

This ensures that the project remains stable and demonstrates basic CI/CD practices.
